# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

**Please do NOT open public GitHub issues for security vulnerabilities.**

To report a security vulnerability, please contact the maintainers directly via email.

When reporting, please include the following information:

- **Description**: A clear description of the vulnerability
- **Reproduction Steps**: Step-by-step instructions to reproduce the issue
- **Potential Impact**: Assessment of the security impact
- **Suggested Fixes**: Any recommendations for remediation (optional)

## What to Expect

- **Acknowledgment**: You will receive an acknowledgment within 48 hours of your report
- **Progress Updates**: We will provide updates during the investigation process
- **Public Disclosure**: After the vulnerability is resolved, we will coordinate public disclosure. Credit will be given to the reporter upon request

## Scope

This security policy applies to the latest version of the project. Older versions may not receive security updates.

## Security Best Practices

Guidelines for contributors to maintain project security:

- **Keep dependencies updated**: Regularly update dependencies to patch known vulnerabilities
- **Never commit secrets**: Do not commit API keys, passwords, tokens, or other sensitive credentials
- **Use environment variables**: Store sensitive configuration in environment variables, not in code
- **Follow principle of least privilege**: Request only the minimum permissions necessary

Thank you for helping keep this project secure!
